LAP Support Ticket Module

Short documentation for permissions, tenant behavior, user flow, manager flow, APIs, and testing.

Purpose

The Support Ticket module lets employees raise issues, track their own ticket status, and lets higher-permission users resolve, update, close, and reopen tickets.

Permissions

PermissionAccess
raise_support_ticketCan raise a new support ticket.
view_support_ticketsCan view My Tracking for own tickets only. Read-only.
manage_support_ticketsCan view all tickets and update, assign, resolve, close, or reopen.
manage_support_ticket_typesCan add or disable dynamic issue types.

Frontend Flow

  • Sidebar item: Support Tickets, shown only with support-ticket permission.
  • Raise Ticket: select issue type, priority, subject, and description.
  • My Tracking: requester sees own ticket status and notes only.
  • All Tickets: manager/admin can update status and add notes.
  • Issue Types: admin/support manager can add or disable issue categories.

Default Issue Types

Technical Issue, Billing, HR Support, Leave Problem, Payroll Issue, Login Problem, Vendor Support, LMS Issue, CRM Issue, Attendance Issue, Asset Request, Network Issue, Software Access, Data Correction, Policy Question.

Status and Priority

TypeValues
StatusOpen, In Progress, Waiting User, Resolved, Closed, Reopened
PriorityLow, Medium, High, Urgent

Tenant Handling

Tenant ID is backend-owned. The frontend does not show or send tenant ID. Support tickets use request.user.tenant_id, so users only work inside their own tenant.

Backend APIs

EndpointPurpose
GET /api/support/ticket-types/List issue types.
POST /api/support/tickets/raise/Raise ticket.
GET /api/support/tickets/my/Requester tracking.
GET /api/support/tickets/all/All ticket list for managers.
POST /api/support/tickets/<id>/action/Manager update/resolve/close/reopen.
GET /api/support/tickets/summary/Permission-based stats.

How To Check

  • Login as employee with raise/view support permissions.
  • Raise a ticket and check My Tracking. It should be read-only.
  • Login as HR/Admin/support manager.
  • Open All Tickets, update status, add notes, resolve or close.
  • Login again as requester and verify updated status and notes.
